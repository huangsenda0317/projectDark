# 《骰境：信仰轮回》MVP 设计文档

> **类型**：像素风 · 轻肉鸽 · 爬塔 · 骰子收集 · 村庄养成
> **引擎**：Godot 4.6
> **文档版本**：v1.0
> **日期**：2026-04-29
> **目标**：界定最小可行产品范围，提供可执行的开发排期

---

## MVP 设计原则

1. **核心循环优先**：骰子收集 → 棋盘移动 → 战斗 → 获取奖励 → 村庄贡献，这条循环必须完整可玩
2. **垂直切片**：一座塔、一套套装、一个 BOSS，但做到完整可玩
3. **系统做减法**：仅实现 GDD Phase 1 列出的系统，其余留到后续阶段
4. **数据驱动**：所有数值从配置文件加载，便于后期调优
5. **可测试交付**：MVP 完成后可独立运行、可试玩、可收集反馈

## MVP 系统详细设计

### 1. 环形棋盘

```
           [宝藏]
    [怪物]           [商店]

[祈祷]   ★中心★   [事件]

    [精英]           [陷阱]
           [BOSS/上楼]
```

**MVP 规格**：
- 圆环形布局，18-22 格（随机 ±2）
- 格子类型：战斗（40%）、事件（20%）、商店（10%）、祈祷所（10%）、宝藏（8%）、陷阱（7%）、上楼格（1 个）、BOSS 格（1 个）
- 中心为「神坛」——到达上楼格后进入层间结算
- 套圈规则：走完一圈后，已触发格变为「虚空格」（+1 诅咒，跳过）
- 格子初始不可见，踩上才揭示（MVP 不做迷雾美术，用灰色占位）

### 2. 骰子系统

**MVP 面数范围**：D4 / D6 / D8 / D10 / D12

| 面数 | 移动范围 | 嵌入伤害基础 | 掉落起始层 |
|------|----------|-------------|-----------|
| D4 | 1-4 | 1d4 | 1 层 |
| D6 | 1-6 | 1d6 | 1 层 |
| D8 | 1-8 | 1d8 | 1 层 |
| D10 | 1-10 | 1d10 | 4 层 |
| D12 | 1-12 | 1d12 | 4 层 |

**MVP 品质范围**：白 / 蓝 / 紫（不含金色传说）

| 品质 | 词条数 | 掉落层数 |
|------|--------|----------|
| 白色（普通） | 1 条 | 1-3 层 |
| 蓝色（精良） | 2 条 | 1-6 层 |
| 紫色（稀有） | 3 条 | 4-6 层 |

**词条精简**（MVP 每种保留 3-4 个核心词条）：

投掷词条：
- 步数修正：投掷后点数 ±1（白+）
- 先手之骰：踩战斗格时先手 +1（蓝+）
- 疾走：顺时针移动时额外 +1 步（紫+）

嵌入词条：
- 锋锐：嵌入武器时暴击率 +5%（白+）
- 坚韧：嵌入防具时护甲 +2（白+）
- 灼热：攻击附带骰子面数×0.5 的火焰伤害（蓝+）
- 吸血：造成伤害的 10% 转为 HP（紫+）

通用词条：
- 治愈之光：每层开始恢复 3 HP（白+）
- 虔诚：信仰获取 +1（蓝+）
- 轻负：负重上限 +2kg（蓝+）

**骰子匣**：
- 初始 2 格
- BOSS 掉落可低概率扩容至 3-4 格（MVP 上限 4 格）
- 匣空兜底：村庄 NPC 免费领 D4 白骰

**骰子分配规则**：
- 嵌入骰子持续生效，需主动取回
- 嵌入骰子也可参与移动投掷（不脱离装备）
- 非战斗可随时重分配；战斗完全锁定
- 嵌入消耗信仰 = 骰子面数 ÷ 3

### 3. 格子类型

| 格子 | 触发效果 | MVP 实现复杂度 |
|------|----------|---------------|
| ⚔️ 战斗 | 随机普通怪物，胜利三选一奖励 | 中 |
| 🏪 商店 | 购买装备/骰子/消耗品（金币） | 低 |
| 🕯️ 祈祷所 | 获得 2-4 信仰 | 低 |
| 📦 宝藏 | 随机获得装备/遗物/金币 | 低 |
| ❓ 事件 | 文字二选一事件（MVP 5 个事件） | 中 |
| 🔥 陷阱 | 受到伤害或负面状态 | 低 |
| ⬆️ 上楼格 | 进入下一层 | 低 |
| 🗡️ BOSS 格 | 触发层 BOSS 战 | 高 |

**MVP 随机事件（5 个）**：

| 事件名 | 选项 A | 选项 B |
|--------|--------|--------|
| 十字路口的流浪汉 | 给他面包（-1 金币，+2 信仰） | 无视（-1 信仰） |
| 异端的小册子 | 焚毁（+3 信仰） | 阅读（-2 信仰，获得随机消耗品） |
| 倒塌的神坛 | 修复（-20 金，+4 信仰） | 取走遗物（获得遗物，-3 信仰） |
| 临死的骑士 | 救治（-5 HP，获得随机装备） | 问询（揭示本层 3 个格子） |
| 神明的低语 | 顺从（随机正面效果） | 质疑（-2 信仰，+15 金币） |

### 4. 战斗系统

**战斗流程**：
```
踩战斗格 → 骰子分配确认（15 秒） → 进入战斗
  ↓
每回合：敌意阶段（显示意图） → 行动阶段（消耗 AP） → 结算 → 磨损结算
  ↓
一方全灭或逃跑 → 战斗结束 → 骰子分配保持不变
```

**行动力（AP）**：
- 每回合 3 AP
- 普攻 1 AP / 副手攻击 1 AP / 防御 1 AP / 技能 2 AP / 道具 1 AP

**骰子伤害公式**：
```
伤害 = 投 Y 面骰 X 次求和 + Z，暴击率 C%，波动 ±R

X = 武器基础（骑士重锤 = 2）
Y = 嵌入骰子面数
Z = 武器基础加值 + STR 加成（每 5 点 STR +1）
C = 武器基础暴击 + AGI 加成（每 10 点 +1%）
R = 武器品级决定
```

**属性影响**：
| 属性 | 初始 | 战斗影响 |
|------|------|----------|
| STR | 3 | 物理伤害 Z 值（每 5 点 +1）；负重上限（每点 +2kg） |
| AGI | 3 | 每 5 点 AP+1；每 10 点暴击 +1% |
| INT | 3 | MVP 暂不绑定法术（留给修士套装） |
| CON | 3 | 初始 HP = 30 + CON×5 = 45；升级 HP 增量 |
| FAI | 3 | 信仰获取效率；信仰技能加成 |

**敌人意图系统**（MVP 3 种意图）：
| 意图 | 说明 | 玩家应对 |
|------|------|----------|
| ⚔️ 攻击 | 造成 X 点伤害 | 防御叠护盾 |
| 🛡️ 防御 | 获得护盾 | 使用道具或等待 |
| 🔥 强力攻击 | 高伤害，需 1 回合蓄力 | 必须防御 |

**骰子磨损**：
- 每次攻击/防御，嵌入骰子磨损 +1
- 磨损上限 = 骰子面数
- 50% 磨损 → 公式结果 -1；75% → -2
- 100% → 碎裂（词条失效，仍可移动）
- 每层结束所有骰子磨损 -2
- 修复方式：铁匠铺（5-15 金币）、修复膏（磨损 -5）

**状态效果**（MVP 精简）：
| 状态 | 效果 | 持续 |
|------|------|------|
| 祝福 | 骰子公式 Y 面数等效 +2 | 3 回合 |
| 诅咒印记 | 每回合受到层数×1 伤害 | 5 回合 |
| 轻灵 | 本回合 +2 AP | 1 回合 |
| 沉重 | 本回合 AP-1（最低 1） | 1 回合 |

### 5. 骑士套装

> MVP 唯一完整实现的套装

| 散件 | 基础属性 | 骰子赋能 | 重量 |
|------|----------|----------|------|
| 骑士重锤 | 伤害 12-18, 暴击 8%, 波动 ±3, X=2 | `2dY+5(8)[-3,3]` | 5kg |
| 骑士胸甲 | +3 护甲, STR+1 | 护甲 +骰子面数（2 回合） | 6kg |
| 骑士头盔 | +2 护甲 | 护盾 +骰子面数 | 3kg |
| 骑士腿甲 | +2 护甲 | 免疫击退（本回合） | 4kg |
| 骑士护腕 | +1 护甲 | 护盾 +骰子面数÷2 | 2kg |
| 骑士盾牌 | 格挡 3 | 护盾 = 骰子面数×2 | 3kg |

| 件数 | 套装效果 |
|------|----------|
| 2 件 | 每回合自动获得 4 点护甲 |
| 4 件 | 解锁「神圣冲锋」：3 回合冷却，造成 STR×3 + 武器骰子面数 伤害 |
| 6 件 | 受到致命伤害时 1 次免死保留 1 HP（每场战斗 1 次） |

总重：23kg（需 STR≥7 → 上限 24kg）

### 6. 属性与升级

**初始属性**：STR 3 / AGI 3 / INT 3 / CON 3 / FAI 3
**初始 HP**：30 + CON(3)×5 = 45
**初始负重上限**：10 + STR(3)×2 = 16kg

**升级经验**：
| 等级 | 累计经验 | 属性点 | HP 增量 |
|------|----------|--------|---------|
| 1 | 0 | — | 基础 45 |
| 2 | 20 | +2 | +5 |
| 3 | 50 | +2 | +5 |
| 4 | 100 | +2 | +8 |
| 5 | 180 | +2 | +8 |

> MVP 等级上限 5（单座塔 3-6 层足够）

### 7. 负重系统

| 状态 | 条件 | 效果 |
|------|------|------|
| ✅ 轻装 | ≤50% 上限 | 闪避 +10%，骰子结果 +1（每层 1 次） |
| ⚖️ 正常 | 50%-80% | 无 |
| ⚠️ 超重 | 80%-100% | 先手概率 -20% |
| ❌ 严重超重 | >100% | 每格 2 点疲劳伤害，伤害 -30% |

背包初始 6 格，每格 1kg 负重计入。

### 9. 圣堂废墟（MVP 唯一塔）

| 属性 | 值 |
|------|-----|
| 塔等级 | Lv.1 |
| 层级数 | 随机 3-6 层 |
| 怪物属性倍率 | ×0.8 |
| 敌人族群 | 堕落修士、圣像守卫 |
| 特殊规则 | 信仰获取翻倍，商店禁用（仅战斗掉落+宝箱获取装备） |
| BOSS | 腐败审判官 |

**腐败审判官（BOSS）**：
- HP：120
- 阶段 1（HP>50%）：每回合对玩家施加 1 层诅咒印记，普攻伤害 8-12
- 阶段 2（HP≤50%）：攻击附带「信念崩塌」（信仰值-2/回合），伤害提升至 12-16
- 掉落：蓝骰 ×1，金币 20，低概率骰子匣 +1

### 10. 死亡与复活

| 惩罚 | 说明 |
|------|------|
| 金币 | 失去当前携带的 30%，遗留在死亡格 |
| 信仰 | 失去当前的 30%，不可找回 |
| 装备 | 15% 概率掉落 1 件，遗留在死亡格 |
| 骰子 | 所有嵌入骰子直接碎裂 |

复活流程：死亡 → 自动回村 → HP 恢复 50%，信仰恢复至 5 → 重新出发 → 死亡格拾取遗留物

> 连续死亡 3 次不掉落装备。

### 关键 AutoLoad

| 单例 | 职责 |
|------|------|
| `EventBus` | 全局信号（`dice_rolled`, `tile_triggered`, `combat_started`, `player_died` 等） |
| `GameState` | 当前 Run 状态（骰子匣、装备、属性、信仰、HP、负重、所在层/格） |
| `SaveManager` | 自动存档/读档（FileAccess + JSON） |
| `RNG` | 种子随机数生成器（`RandomNumberGenerator` 封装） |
