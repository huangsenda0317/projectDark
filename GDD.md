## 1. 游戏概述

### 一句话描述

> 在一个诅咒的中世纪轮回之境中，你是一位信仰者兼村长——收集并驾驭骰子之灵，在环形棋盘上闯关，将骰子嵌入装备释放真正力量，带着战利品回乡建设村庄，让信仰在大地与高塔间流转。

### 游戏简介

《骰境：信仰轮回》是一款**单人回合制像素风轻肉鸽游戏**，使用 **Godot 4.6** 引擎开发。玩家扮演一位兼任村长的信仰者，在**环形棋盘**上通过**投掷骰子**移动，触发随机格子事件，以战斗、祈祷、交易与抉择逐层爬上「神之塔」。

骰子是本游戏的**唯一核心道具**——不再是一次性消耗品，而是可收集、有品质、带词条的实体宝物。骰子同时驱动三个系统：**移动（投掷步数）、战斗（嵌入装备释放骰子公式）、探索（触发特殊格子）**。

游戏拥有**双主线结构**：

- **爬塔线**：在塔中探索，收集稀有骰子嵌入装备，通过**骰子驱动的 Build 系统**（无固定职业）自由定制角色风格
- **养成线**：将塔中获取的资源与信仰值贡献给村庄，发展村落，获得爬塔增益

**核心关键词**：骰子收集 · 环形棋盘 · 骰子嵌入赋能 · 负重管理 · 信仰双线 · 村庄养成 · 像素美学 · 多主题换皮

## 2. 核心体验目标

| 目标             | 说明                                                 |
| ---------------- | ---------------------------------------------------- |
| **每局不同**     | 棋盘格子随机生成，每次爬塔路径都是新体验             |
| **有意义的选择** | 骰子分配策略、方向选择、装备搭配、何时卸甲减重       |

## 3. 世界观与美术风格

### 3.1 世界观

时代背景是一个**架空中世纪欧洲**，正统教会正在衰落，神迹被质疑，异端横行。  
玩家踏入的「神之塔」是一座古老的试炼之地：

- 传说中完成试炼者可获得「真理之言」——神明真正的意志
- 每上升一层，世界观念越模糊——究竟何为信仰？何为盲从？
- 故事无固定结局，结局由玩家的信仰值与选择决定

**主题对立**：

- 秩序 vs 混沌
- 圣洁 vs 亵渎
- 服从 vs 自由意志

### 3.2 美术风格

- **分辨率**：320×180 基础分辨率（像素放大 ×3~4 显示）
- **调色盘**：暖黄/羊皮纸底色 + 深蓝/紫色阴影，点缀圣光金
- **参考风格**：Unicorn Overlord 地图格 + Darkest Dungeon 暗调 + Moonlighter 明亮人物像素
- **角色设计**：16×16 像素小人，战斗时 32×32 放大动画
- **棋盘**：俯视角石板地图，骨骼纹理，有蜡烛、苔藓等装饰细节
- **UI 框架**：羊皮纸卷轴风格，哥特字体，金属装饰边框

### 3.3 多主题换皮系统

「骰境」系列支持主题切换，玩家在主界面选择主题即可整体换皮。核心玩法不变，视听层全面替换。

| 主题 | 代号 | 骰子材质 | 棋盘风格 | 敌人风格 | BGM 基调 |
|------|------|----------|----------|----------|----------|
| **中世纪地下城** | 信仰轮回 | 骨/石/金属 | 石板纹理 | 骑士/修士/异端 | 管风琴 + 圣歌 |
| **中国洪荒** | 洪荒纪 | 玉石/青铜/兽骨 | 太极云纹 | 妖兽/巫族/散仙 | 古筝 + 鼓乐 |
| **中国仙侠** | 剑仙问道 | 灵石/仙玉/剑魄 | 灵山仙雾 | 妖魔/剑修/邪道 | 笛箫 + 编钟 |
| **日本百鬼** | 百鬼夜行 | 勾玉/符纸/妖骨 | 神社鸟居 | 妖怪/阴阳师/鬼族 | 三味线 + 太鼓 |

**主题切换规则**：
- 主界面选择主题，切换后全 Run 生效
- 不同主题的骰子外观不同，但品质/面数/词条机制完全一致
- 敌人名称、事件文本、UI 边框同步切换
- 村庄建筑外观随主题变化
- 未来可扩展 Steam 创意工坊，开放玩家自制主题

## 4. 技术基础

### 4.1 引擎选型：Godot 4.x

| 方面         | 说明                                                               |
| ------------ | ------------------------------------------------------------------ |
| **引擎版本** | Godot 4.6（详见 docs/engine-reference/godot/VERSION.md）             |
| **渲染管线** | 2D 像素渲染，关闭抗锯齿，使用 `CanvasItem` 着色器实现像素发光/滤镜 |
| **场景结构** | 主场景 → 棋盘场景 / 战斗场景 / 村庄场景（通过 SceneTree 切换）     |
| **数据管理** | 装备/遗物/道具数据使用 `Resource`（`.tres`）定义，便于编辑器配置   |
| **存档系统** | `FileAccess` + JSON 序列化，支持单槽自动存档                       |
| **信号系统** | 全局 `EventBus`（AutoLoad 单例），解耦各模块通信                   |
| **随机系统** | `RandomNumberGenerator` 配合种子，支持 Run 种子分享功能            |

### 4.2 项目结构建议

```
res://
├── scenes/
│   ├── board/        # 棋盘场景
│   ├── combat/       # 战斗场景
│   ├── village/      # 村庄养成场景
│   ├── ui/           # HUD、菜单、奖励选择UI
│   └── world/        # 主世界/入口场景
├── scripts/
│   ├── systems/      # 核心系统（骰子/信仰/负重/Build）
│   ├── entities/     # 玩家、敌人、NPC
│   └── autoloads/    # EventBus, GameState, SaveManager
├── resources/
│   ├── equipment/    # 装备数据 .tres
│   ├── items/        # 道具数据 .tres
│   ├── relics/       # 遗物数据 .tres
│   └── enemies/      # 敌人数据 .tres
├── assets/
│   ├── sprites/      # 像素图资源
│   ├── audio/        # 音效/BGM
│   └── fonts/        # 哥特像素字体
└── data/
    └── events/       # 随机事件 JSON 数据
```

### 4.3 关键 Godot 特性应用

- **TileMap**：村庄建设地图使用 TileMap 管理建筑摆放
- **AnimationPlayer**：骰子投掷动画、棋子移动动画
- **ShaderMaterial**：装备发光特效（骑士套装金光、异端套装紫焰）
- **SubViewport**：战斗场景嵌入棋盘 UI，避免全场景切换
- **GDScript**：主要脚本语言，关键数值计算模块可用 C# 提速

## 5. 核心机制

### 5.1 游戏流程（宏观）

```
┌─────────────────────────────────────────────────────────┐
│                   一次 Run 的宏观流程                     │
│                                                         │
│  村庄大厅 → 进入地图室，查看世界地图                      │
│      ↓                                                  │
│  选择一座已解锁的塔 → 消耗旅途补给 → 出发                │
│      ↓                                                  │
│  进入塔 → 塔门准备（分配骰子到装备/备用，装备调整）       │
│      ↓                                                  │
│  ┌── 塔内循环 ──────────────────────────┐               │
│  │                                       │               │
│  │  新层开始 → 环形棋盘生成               │               │
│  │      ↓                                │               │
│  │  非战斗格：可随时重分配骰子            │               │
│  │      ↓                                │               │
│  │  选择骰子投掷 → 顺/逆时针移动           │               │
│  │      ↓                                │               │
│  │  触发格子事件                          │               │
│  │      ├─ 战斗格 → 进入战斗（骰子锁定）  │               │
│  │      └─ 非战斗格 → 结算后可重分配      │               │
│  │      ↓                                │               │
│  │  层间结算（神坛 → 继续上升/绕圈刷）     │               │
│  │      ↓                                │               │
│  │  击败 BOSS → 塔通关                    │               │
│  └──────────────────────────────────────┘               │
│      ↓                                                  │
│  返回村庄 → 贡献资源 → 结算                              │
│      ↓                                                  │
│  选择：继续探索下一座塔 or 结束 Run                      │
└─────────────────────────────────────────────────────────┘
```

### 5.2 塔内一层流程

```
┌─────────────────────────────────────────────────────────┐
│                      一层流程                            │
│                                                         │
│  开始新层 → 环形棋盘生成（18~36格，随塔等级和层数）       │
│      ↓                                                  │
│  选择骰子投掷 → 选择方向（顺/逆时针）→ 移动              │
│      ↓                                                  │
│  触发格子事件                                           │
│      ├─ 战斗格 → 进入战斗（骰子锁定至战斗结束）          │
│      ├─ 非战斗格 → 结算后骰子可自由重分配                │
│      └─ BOSS格 → 进入BOSS战（骰子锁定至战斗结束）        │
│      ↓                                                  │
│  在棋盘上循环，直到踩中进入下一层或BOSS格子               │
│      ↓                                                  │
│  层间结算 → 上升到下一层（棋盘重新生成）                  │
└─────────────────────────────────────────────────────────┘
```

### 5.3 行动基本规则

- 每回合从**所有已拥有的骰子中选择一枚投掷**，点数即移动步数
  - 嵌入在装备中的骰子**也可以参与移动投掷**（骰子不会从装备上取下）
  - 除非获得「多骰投掷」类道具，否则每回合只能选**一枚骰子**投掷
- 玩家选择**顺时针或逆时针**移动
- 落点格子**必须触发**，无法跳过
- 某些遗物/道具可以**修改骰子结果**（+1/-1/重投/固定点数等）
- 环形棋盘可以**套圈**（高点数绕多圈），每多走一圈「诅咒槽」+1
- 进入塔内所有格子**初始不可见**，踩上才知道内容
- **非战斗状态下**，玩家可随时打开骰子分配界面，调整骰子的嵌入/取回状态

### 5.4 骰子分配核心规则（v0.4 新增）

> 骰子分配是本游戏最核心的策略维度之一。分配好了就不会自动改变，直到玩家自己重新分配。

```
┌─────────────────────────────────────────────────────────┐
│                骰子分配状态机                             │
│                                                         │
│  骰子有三种状态：                                       │
│                                                         │
│  ┌──────────┐   嵌入装备   ┌──────────┐                │
│  │  自由状态  │ ──────────→ │  嵌入状态  │                │
│  │ (未嵌入    │ ←────────── │ (锁定在    │                │
│  │  装备)    │  取回嵌入    │  装备槽位) │                │
│  └─────┬────┘              └──────────┘                │
│        │                                                 │
│        │  战斗格触发时                                    │
│        ↓                                                 │
│  ┌──────────┐                                           │
│  │  战斗锁定  │ ← 嵌入状态在战斗中不可更改               │
│  │ (战斗期间  │                                           │
│  │  不可重分配)│ → 战斗结束后回到原来的嵌入状态            │
│  └──────────┘                                           │
│                                                         │
│  重分配时机：                                             │
│  ✅ 非战斗格子事件结算后                                  │
│  ✅ 层间神坛                                             │
│  ✅ 进入塔门准备时                                       │
│  ❌ 战斗进行中                                           │
│                                                         │
│  移动投掷：                                              │
│  ✅ 自由骰子 → 选择投掷，点数 = 步数                     │
│  ✅ 嵌入骰子 → 同样可选中投掷（不脱离装备）              │
│  ❌ 碎裂骰子 → 可投掷移动，但词条失效、嵌入效果无效      │
└─────────────────────────────────────────────────────────┘
```

**关键设计点**：
- 骰子一旦嵌入装备，**持续生效**，不会自动脱落——玩家需要主动取回
- **嵌入骰子同样可以用于移动投掷**——投掷不改变嵌入状态，骰子留在装备上
- 移动时从所有骰子（自由+嵌入）中选择一枚投掷，除非有道具允许投掷多枚
- 战斗结束后骰子**不会自动脱离**——分配保持不变，玩家主动决定是否调整

## 6. 骰子系统（核心道具）

> **核心设计理念**：骰子不再是消耗性随机数生成器，而是可收集、有品质、带词条的实体宝物。它是本游戏的唯一核心道具，同时驱动**移动**、**战斗**、**探索**三大系统。玩家的一切决策都围绕骰子展开。

### 6.1 骰子概述

一枚骰子同时服务于三个维度：

```
            ┌──────────────┐
            │   骰子实体    │
            │ (品质+词条)   │
            │ (面数D4~D100) │
            └───┬───┬───┬──┘
           ┌────┘   │   └────┐
           ↓        ↓        ↓
      棋盘移动    装备嵌入   格子触发
     (投掷步数)  (释放公式) (特殊事件)
```

- **移动**：从所有骰子中选择一枚投掷，点数 = 步数（嵌入中的骰子也可投掷，不脱离装备）
- **嵌入**：将骰子嵌入装备骰槽，激活装备的骰子伤害公式和词条加成（持续生效，需主动取回）
- **触发**：某些特殊格子需要特定品质/面数的骰子来解锁

### 6.2 骰子品质与词条

| 品质 | 稀有度色 | 词条数 | 掉落层数 | 说明 |
|------|----------|--------|----------|------|
| **普通** | 白 | 1 条 | 1-3 层 | 常见，基础词条 |
| **精良** | 蓝 | 2 条 | 1-6 层 | 较常见，实用词条 |
| **稀有** | 紫 | 3 条 | 4-10 层 | 强力词条组合 |

### 6.5 骰子匣

骰子匣是玩家的**骰子携带容器**，初始容量极小，扩容是核心成长线。

| 属性 | 初始值 | 上限 |
|------|--------|------|
| 骰子匣格子 | **2 格** | 10 格 |
| 初始骰子 | **1×D4（白）+ 1×D6（白）** | — |

> v0.4 变更说明（v0.4.1 更新）：骰子匣格子从 3 恢复为 2。由于嵌入骰子也可以参与移动投掷，不再需要额外预留"备用骰子"，匣格数量反映的是骰子**持有总量上限**。

**扩容方式**：

| 方式 | 条件 | 说明 |
|------|------|------|
| BOSS 掉落 | 每层 BOSS 击杀后，三选一奖励中**低概率**出现 +1 格 | 主要来源 |
| 商店购买 | 高价（50 金币 / 格，每 Run 限购 1 次） | 补充来源 |
| 村庄功能 | 村庄铁匠相关功能可永久 +1 格（跨 Run） | 永久进度 |

**兜底机制**：
- 骰子匣为空（所有骰子都碎裂且无修复手段）时，出发前可向村庄 NPC **免费领取 1 枚 D4 白色骰子**
- 塔内商店**固定出售 D4 骰子**（10 金币），确保不会软锁

### 6.6 骰子用途一：移动（棋盘骰子）

每回合从**所有已拥有的骰子**（自由骰子 + 嵌入骰子）中选择一枚投掷，点数 = 移动步数。

- **所有骰子均可参与移动投掷**，包括已嵌入装备的骰子（投掷不改变嵌入状态）
- 碎裂的骰子也可投掷移动，但词条失效
- 除非获得「多骰投掷」类道具，每回合只能投掷一枚
- 玩家选择**顺时针或逆时针**移动
- 落点格子**必须触发**，无法跳过
- 环形棋盘可套圈（高点数绕多圈），每多走一圈「诅咒槽」+1
- 投掷后该骰子**留在原位**（自由骰子留在匣中，嵌入骰子留在装备上），下次移动可继续使用

### 6.7 骰子用途二：嵌入赋能（战斗骰子）

参见 [第 9 章 战斗系统](#9-战斗系统骰子嵌入制) 完整说明。简述：

- 将骰子**嵌入装备骰槽**，激活装备的骰子伤害公式和词条
- 嵌入消耗信仰值 = **骰子面数 ÷ 3**（D4=1, D6=2, D8=2, D10=3, D12=4, D20=6, D100=33）
- 嵌入后骰子**持续锁定在装备上**，不会自动脱落——玩家需主动取回
- 取回嵌入骰子**不退还信仰值**
- 非战斗状态下可随时重分配；战斗中完全锁定
- **嵌入骰子仍可参与移动投掷**（投掷时骰子不脱离装备）
- 武器嵌入后获得骰子伤害公式（如 `1d6+5`）；防具嵌入后获得护盾/护甲公式

> v0.4 变更：嵌入消耗从「面数÷2」调整为「面数÷3」，降低嵌入门槛，鼓励玩家多尝试不同分配方案。

## 9. 战斗系统（骰子嵌入制）

> **核心设计理念**：战斗采用骰子嵌入制——玩家在非战斗状态下分配骰子到装备槽位，踩到战斗格时骰子分配锁定，进入战斗。战斗中每回合选择攻击/防御/技能，消耗行动力。骰子有磨损机制，长时间使用同一嵌入骰子会有衰减风险。策略重心在于「如何分配骰子」以及「战斗中的行动力管理」。

### 9.1 战斗概述

战斗采用**骰子嵌入制**（Dice Embedding）：

- **踩到战斗格时**：当前骰子分配状态锁定，进入战斗
- **战中**：每回合拥有**行动力**（AP），选择消耗 AP 执行行动，伤害由已嵌入的骰子公式自动驱动
- **骰子磨损**：嵌入的骰子在战斗中使用会累积磨损，磨损过高时骰子性能衰减
- **战斗结束**：骰子分配保持不变（不会自动归还匣中），磨损部分恢复
- **敌人意图系统**：保留，敌人在玩家行动前预告本回合行动

**核心决策点（分配阶段——非战斗状态）**：
- 这颗 D12 嵌入武器换高伤害，还是留着作为移动时的高点数骰子？
- 多颗骰子分别嵌入哪些装备——每颗骰子的嵌入/自由状态都影响移动选择
- D100 嵌入哪个槽位能最大化传奇词条收益？

**核心决策点（战斗中）**：
- 有限的行动力分配：全力攻击还是攻防兼顾？
- 骰子磨损在累积——是继续用嵌入的强力骰子，还是切换策略？
- 看到敌人蓄力 → 本回合用防御（嵌入防具的骰子提供护盾）
- 血量危险 → 使用道具或技能

### 9.2 行动力系统（v0.4 新增）

每回合玩家拥有**行动力（Action Points, AP）**，用于执行各种行动：

| 属性 | 初始值 | 成长方式 |
|------|--------|----------|
| 每回合 AP | **3 点** | 敏捷属性每 5 点 +1 AP；部分装备/遗物加成 |

**行动 AP 消耗**：

| 行动 | AP 消耗 | 说明 |
|------|---------|------|
| 普攻 | 1 AP | 使用主手武器攻击 |
| 副手攻击 | 1 AP | 使用副手武器攻击（需副手嵌入骰子） |
| 防御 | 1 AP | 获得护盾/护甲 |
| 技能 | 2 AP | 使用套装技能或装备技能 |
| 道具 | 1 AP | 使用消耗品 |
| 移动（战斗中） | 1 AP | 战术中调整站位（仅部分战斗场景可用） |

> 行动力系统的设计目标：增加战斗中的决策密度。玩家不再只是「攻/防/技能」三选一，而是需要在有限 AP 内组合多个行动——比如「攻击+防御」（2AP）或「技能+道具」（3AP）。

### 9.3 骰子磨损系统（v0.4 新增）

嵌入装备的骰子在战斗中会累积**磨损值**：

| 机制 | 说明 |
|------|------|
| 磨损触发 | 每次使用嵌入骰子的装备执行攻击/防御，磨损 +1 |
| 磨损阈值 | 每颗骰子的磨损上限 = 骰子面数（D6 上限 6，D20 上限 20） |
| 磨损效果 | 磨损达到上限的 50% 时，骰子公式结果 -1；达到 75% 时 -2 |
| 骰子碎裂 | 磨损达到 100% 时骰子**碎裂**——从装备槽位脱落，词条失效，嵌入效果无效 |
| 碎裂后状态 | 碎裂骰子**仍可参与移动投掷**（点数正常计算），但所有词条不生效 |
| 磨损恢复 | 每层结束后所有骰子磨损 -2 |

#### 骰子修复方式

碎裂或磨损较高的骰子可以通过以下方式修复：

| 修复方式 | 消耗 | 效果 | 获取渠道 |
|----------|------|------|----------|
| **铁匠铺修复** | 5~15 金币 | 碎裂骰子完全修复；或指定骰子磨损归零 | 村庄铁匠铺（村庄等级≥2） |
| **修复膏** | 1 枚消耗品 | 单颗骰子磨损 -5（不修复碎裂） | 塔内商店/村庄商店（15 金币） |
| **复活圣水** | 1 枚消耗品 | 单颗碎裂骰子恢复为完好状态 | 祈祷所/BOSS 掉落/村庄商店（30 金币） |
| **磨损护符（遗物）** | 持有即生效 | 所有骰子磨损速度 -1（最低 0） | 精英/BOSS 掉落 |

**磨损的策略意义**：
- 长时间战斗（BOSS 战）需要考虑磨损——一味普攻可能导致骰子碎裂
- 鼓励玩家在战斗中轮换使用不同装备（而不是只用最强武器）
- 高面数骰子（D20/D100）更耐磨损，是长线战斗的优势
- 低面数骰子（D4/D6）磨损快但嵌入消耗低——适合短平快的普通战
- 碎裂后骰子仍可用于移动，不会让玩家完全"瘫痪"，但战斗能力大打折扣

### 9.4 战前确认：骰子分配检查

### 9.5 回合流程

```
┌─────────────────────────────────────────────────────────┐
│                    一回合战斗流程                         │
│                                                         │
│  ① 敌意阶段：敌人显示本回合意图（攻击/防御/技能）       │
│      ↓                                                  │
│  ② 行动阶段：玩家消耗 AP 执行行动（可多行动）            │
│      ↓                                                  │
│  ③ 结算阶段：按速度排序结算所有行动                      │
│      ↓                                                  │
│  ④ 磨损结算：已使用的嵌入骰子磨损 +1                    │
│      ↓                                                  │
│  ⑤ 回合结束：清除临时状态，AP 恢复，进入下一回合         │
│      ↓                                                  │
│  重复直到一方全灭或逃跑                                  │
└─────────────────────────────────────────────────────────┘
```

**玩家可用行动**：

| 行动 | 说明 | 条件 |
|------|------|------|
| 普攻 | 使用主手武器攻击，伤害由嵌入骰子公式计算 | 需主手武器嵌入骰子，1 AP |
| 副手攻击 | 使用副手武器攻击 | 需副手武器嵌入骰子，1 AP |
| 防御 | 获得护盾/护甲，数值由嵌入防具的骰子公式计算 | 需防具嵌入骰子，1 AP |
| 技能 | 使用套装技能或装备技能 | 2 AP，部分需要骰子嵌入 |
| 道具 | 使用消耗品（不消耗骰子，直接使用） | 背包中持有，1 AP |

### 9.6 骰子伤害公式（继承 Elona 式）

伤害公式沿用 **XdY+Z(C)[±R]** 格式，其中 X, Y 由**嵌入的骰子**决定：

```
伤害 = 投 Y 面骰 X 次求和 + Z，暴击率 C%，最终波动 ±R
```

| 符号 | 含义 | 由什么决定 |
|------|------|-----------|
| **X** | 骰子数量 | 武器基础（1）+ 信念加成 |
| **Y** | 骰子面数 | **嵌入骰子的面数**（D6→6, D12→12, D100→100） |
| **+Z** | 固定加值 | 武器基础加值 + 属性加成 |
| **(C)** | 暴击率 | 武器基础暴击 + 敏捷加成 + 骰子词条 |
| **[±R]** | 最终波动 | 武器品级决定 |

#### 武器嵌入示例

| 武器 | 基础属性 | 嵌入骰子 | 激活伤害公式 | 说明 |
|------|----------|----------|-------------|------|
| 长剑 | 加值+3, 暴击5%, 波动±2 | D6（白） | `1d6+3(5)[-2,2]` | 早期标准配置 |
| 长剑 | 加值+3, 暴击5%, 波动±2 | D12（紫） | `1d12+3(5)[-2,2]` | 中期大幅升级 |
| 骑士重锤 | 加值+5, 暴击8%, 波动±3, X=2 | D20（金） | `2d20+5(8)[-3,3]` | 后期终极配置 |
| 修士法器 | 加值+8, 暴击3%, 波动±1 | D100（金） | `1d100+8(3)[-1,1]` | 传说骰，极端波动 |

#### 属性对公式的影响

| 属性 | 影响 |
|------|------|
| **力量（STR）** | 物理武器：每 5 点力量，固定加值 Z+1 |
| **智力（INT）** | 法术/信仰技能：每 5 点智力，伤害骰面数 Y 等效+1 |
| **敏捷（AGI）** | 每 5 点敏捷，每回合 AP+1；每 10 点敏捷，暴击率 C+1% |
| **信念（FAI）** | 信仰技能：每 5 点信念，骰子数量 X+1（上限+2） |

> v0.4 变更：AGI 的行动优先级机制整合进 AP 系统——AGI 越高，每回合可用 AP 越多，自然行动越"优先"。

### 9.7 敌人意图系统

### 9.8 战斗特色：棋盘骰子联动

部分技能与**本局棋盘骰子点数**挂钩：

- 「命运翻转」：将本层已用棋盘骰子点数总和转化为首回合额外伤害

## 10. 装备与 Build 系统

> **核心设计理念**：游戏没有固定职业。玩家通过收集和穿戴装备，自由组合出战斗风格。装备套装触发专属技能，形成类职业的体验，但完全由玩家自主选择。

### 10.1 属性系统

| **智力（INT）** | 增加法术/信仰技能伤害；增加技能效果数值              | **3**  |
| **体质（CON）** | 增加最大 HP；提升护甲值上限                          | **3**  |
| **信念（FAI）** | 提高信仰值获取效率；信仰技能加成（与信仰系统深度联动）  | **3**  |

**属性成长规则**：

| 成长来源 | 说明 | 上限 |
|----------|------|------|
| **升级手动分配** | 每次升级获得 **2 点属性点**，玩家自由分配到任意属性 | 单属性上限 20 |

| **套装效果** | 套装激活后的被动属性加成 | 套装限定 |

**属性果实**（塔内随机掉落）：

| 果实 | 效果 | 掉落来源 |
|------|------|----------|
| 力量之果 | STR +1 | 精英战/BOSS |

|------|----------|-----------|---------|

| 2 | 20 | +2 | +5 |
| 3 | 50 | +2 | +5 |
| 4 | 100 | +2 | +8 |
| 5 | 180 | +2 | +8 |
| 6+ | 递增 | +2/级 | +8/级 |

### 10.2 装备槽位

```
┌────────────────────────────────────────┐
│  头盔   │  胸甲   │  护腕   │  腿甲    │
│  武器   │  副手   │  戒指×2 │  项链    │
│  披风   │  腰带   │  靴子   │          │
└────────────────────────────────────────┘
```

- 共 **11 个槽位**（戒指占 2 格）
- 每件装备有**重量值（kg）**
- 玩家**负重上限 = 基础 10kg + 力量 ×2kg**（初始 STR=3 → 上限 16kg）
- 每件装备有一个**骰槽**（Dice Socket），可嵌入一枚骰子

### 10.3 装备基础属性与骰子赋能

装备在**嵌入骰子之前**仅提供基础属性，嵌入骰子后释放其真正力量：

```
装备 = 基础属性（固定） + 骰子赋能（由嵌入骰子决定）

┌──────────────────────────────────────────┐
│  长剑（基础）                              │
│  基础伤害：5-8                             │
│  基础暴击：5%                              │
│  重量：3kg                                 │
│                                            │
│  骰槽：[ 空 ]  ← 嵌入骰子激活骰子公式      │
│  ─────────────────────────────             │
│  嵌入 D6  → 伤害公式 1d6+3(5)[-2,2]        │
│  嵌入 D12 → 伤害公式 1d12+3(5)[-2,2]       │
│  嵌入 D20 → 伤害公式 1d20+3(5)[-2,2]       │
└──────────────────────────────────────────┘
```

**装备部件与骰子赋能效果**：

| 装备部位 | 基础属性 | 嵌入骰子后 |
|----------|----------|-----------|
| 武器 | 固定伤害范围 + 基础暴击率 | 激活骰子伤害公式（XdY+Z(C)[±R]） |
| 防具（胸甲/头盔/腿甲等） | 固定护甲值 + 属性加成 | 激活护盾公式 + 骰子通用词条 |
| 副手（盾牌） | 固定格挡值 | 护盾值 = 骰子面数 × 格挡系数 |
| 饰品（戒指/项链） | 属性 +1~2 | 骰子词条效果翻倍 |

> v0.4 变更：明确武器**不提供属性加成**，属性加成仅来自防具和饰品。

**嵌入规则**：
- 嵌入消耗信仰值 = 骰子面数 ÷ 3（向下取整）
- 每件装备同一时间只能嵌入 1 颗骰子
- 嵌入后持续生效，需主动取回（不退还信仰值）
- 非战斗状态可随时重分配；战斗中完全锁定
- 装备可以留空（骰槽为空），仅享受基础属性

## 11. 负重与背包系统

> 重量是战略资源，不只是限制。

### 11.1 负重机制

```
负重上限 = 10kg（基础）+ 力量属性 × 2kg
当前负重 = 所有已装备物品重量之和 + 背包中携带物品重量
```

**负重状态**：

| 状态        | 条件             | 效果                                       |
| ----------- | ---------------- | ------------------------------------------ |
| ✅ 轻装     | 负重 ≤ 上限 ×50% | 闪避率+10%，骰子结果+1（每层 1 次）        |
| ⚖️ 正常     | 50%~80%          | 无额外效果                                 |
| ⚠️ 超重     | 80%~100%         | 移速动画变慢（演出效果），战斗先手概率-20% |
| ❌ 严重超重 | >100%            | 每踩 1 格受到 2 点疲劳伤害，战斗中伤害 -30% |

### 11.2 背包系统

- 玩家有一个**普通背包**：初始容量 **6 格**，每格 1kg 负重计入
- 背包中的物品（装备/遗物/消耗品）**不在身上时不触发被动效果**
- 可在塔中商店购买背包扩容（+2 格，有上限）

### 11.3 次元空间（Dimensional Pouch）

**次元空间**是一种特殊的空间扩展法术/遗物，提供**不计入负重的额外储存格**：

| 名称     | 获得方式          | 容量           | 特性                          |
| -------- | ----------------- | -------------- | ----------------------------- |
| 次元小袋 | 商店购买/宝藏获得 | +4 格，0 重量  | 基础次元容器                  |
| 异端法阵 | 异端格子事件      | +8 格，0 重量  | 代价：进入时-2 信仰           |
| 神圣容器 | 圣化格奖励        | +6 格，0 重量  | 存放的装备获得+1 圣洁层数     |
| 虚空裂缝 | BOSS 掉落（稀有） | +16 格，0 重量 | 随机：取出物品时有 5%概率消失 |

**次元空间限制**：

- 次元空间中的物品**不影响负重**，但**取用有成本**（消耗 2 点信仰值或特定条件）
- 某些格子事件可能导致次元空间**不稳定**，暂时无法访问
- 村庄仓库可永久存放部分物品（下一次 Run 可携带出发）

### 11.4 管理策略示例

```
场景：玩家背包满了，发现了一件强力的骑士胸甲
选项A：装备它，卸下现有胸甲丢弃
选项B：放入次元空间（消耗 2 点信仰值）
选项C：放入背包，但超重了——接下来几格会受到疲劳伤害
选项D：回村时存入仓库（需先结束本层）
```

## 12. 信仰系统

> 信仰是本游戏最核心的差异化机制——它是一种**纯粹的货币**，在塔内用于嵌入骰子和购买圣物，在村庄中用于建设和招募，两条线共享同一个"信仰池"，形成取舍张力。

### 12.1 信仰值（Faith Points）

> v0.4 重大变更：移除「塔内初始 5 点信仰」概念。信仰值纯粹作为一种货币存在，玩家初始拥有一定量的信仰值，之后通过游戏行为获取和消耗。

- **初始信仰值**：**10 点**（首次进入游戏时）
- **信仰值无上限**——可无限积累，鼓励长期规划和大量储备
- **信仰值不会自动恢复**——它是一种货币，获取后需谨慎使用
- **村庄信仰**：每次 Run 结算后，玩家选择将部分信仰值贡献给村庄（不可撤回）
- **二者权衡**：留在手里的信仰值提升当局嵌入和购买能力；贡献给村庄的信仰推动长期养成

### 12.2 信仰值获取与消耗

**获取方式**：

| 来源 | 信仰值 | 说明 |
|------|--------|------|
| 踩到祈祷所 | +2~4 | 最稳定的塔内来源 |
| "信仰选项"的事件选择 | +1~3 | 做出虔诚选择 |
| 击败亵渎型怪物 | +1 | 特定敌人类型 |
| 轻装状态踩圣化格 | +3 | 需要轻装状态 |
| 装备/遗物被动 | +1~2 | 修士袍等 |
| Run 结算奖励 | +5~15 | 根据表现 |
| 村庄功能回馈 | +1~3 | 村庄贡献后的反哺 |

**消耗方式**：

| 消耗 | 信仰值 | 说明 |
|------|--------|------|
| 骰子嵌入 | 面数÷3 | 核心消耗 |
| 激活套装技能 | 3~8 | 战斗中主动技能 |
| 商店购买圣物 | 3~8 | 特殊道具 |
| 信仰翻面 | 3 | 骰子修正 |
| 次元空间取物 | 2 | 每次取物 |
| 套装散件兑换 | 20 | 指定套装随机散件 |
| 村庄贡献 | 可变 | 推动村庄进度 |

### 12.3 信仰标签系统

玩家积累信仰选择，会获得**标签**，影响全局效果：

| 标签         | 获得条件              | 效果                              |
| ------------ | --------------------- | --------------------------------- |
| **圣战者**   | 单次 Run 内累计信仰获取 ≥ 50 | 攻击亵渎型敌人+50%伤害            |
| **异端者**   | 3 次选择亵渎选项      | 解锁异端道具池，信仰获取效率-20%            |
| **殉道者**   | HP 低于 20%时触发战斗 | 低血量时攻击力+2，但不能回血      |
| **怀疑者**   | 5 次拒绝神明指引      | 所有随机事件结果可重投一次        |
| **虔信者**   | Run 开始时信仰值不低于 10，且全程不低于 0   | 每层开始额外获得一件圣洁道具          |
| **慷慨村长** | 累计贡献信仰值 ≥ 50   | 村庄加成反哺塔内：每层起始+1 信仰 |

## 13. 村庄养成系统

> 玩家是村长，塔是副业，村庄是家。每次冒险都是为了让村子活下去，变得更好。

### 13.1 系统概览

村庄养成是**跨 Run 的永久进度系统**，与爬塔形成**资源循环**：

```
爬塔 → 获取资源（金币/材料/信仰） → 贡献给村庄
                                       ↓
                              村庄升级 → 解锁新功能/NPC
                                       ↓
                              新功能/NPC → 提供爬塔增益
                                       ↓
                              更强地爬塔 → 获取更多资源
```

> v0.4 变更：村庄系统大幅精简。移除税收系统、村庄危机事件、建筑升级子系统。村庄功能聚焦在**贡献-解锁**循环上，避免打断肉鸽心流。

### 13.2 村庄等级（定居点进化）

| 等级 | 名称     | 升级条件                   | 解锁内容                       |
| ---- | -------- | -------------------------- | ------------------------------ |
| 1    | **村落** | 初始状态                   | 基础设施（家、祈祷场、出发点、基础商店） |
| 2    | **村庄** | 贡献资源 ×50 + 信仰 ×20    | 铁匠铺功能、骰子合成、套装散件兑换 |
| 3    | **镇子** | 贡献资源 ×200 + 信仰 ×80   | 教堂功能、斥候派遣、属性果实商店 |
| 4    | **城市** | 贡献资源 ×600 + 信仰 ×200  | 次元研究所、仓库扩容、稀有套装兑换 |
| 5    | **圣城** | 特殊条件（完成真结局分支） | 终极塔「神之眼」出现、永久爬塔增益最大化 |

### 13.3 信仰值在村庄中的功能

信仰值是村庄的**第二货币**，专用于以下功能：

| 功能           | 消耗信仰     | 效果                                    |
| -------------- | ------------ | --------------------------------------- |
| **升级定居点** | 参见上表     | 推进村庄等级                            |

| 功能 | 解锁等级 | 说明 |

## 14. 世界地图与塔系统

> **核心设计理念**：高塔不再唯一，而是林立在世界各地。村庄在地图中央，玩家从地图室出发，选择一座塔进行探索。塔有身份、等级、层级三个独立维度，每次进入随机生成，形成类副本的探索体验。

### 14.1 世界概述

```
                     [异端要塞 Lv.5]
                           │
              [瘟疫地窖 Lv.3]──┼──[钟楼残骸 Lv.4]
                           │
          [圣堂废墟 Lv.1]──┼──[沉默修道院 Lv.2]──[? 未探索]
                           │
                     ★ 村庄（中心）★
                           │
              [? 未探索]──┼──[地窖试炼 Lv.1]
                           │
                     [? 未探索]
```

- **村庄始终在世界中心**
- 塔散布在村庄四方，通过道路连接
- 已发现的塔永久可见，显示名称、等级、状态
- 未探索区域以**迷雾**遮盖，派遣斥候可逐步揭示

### 14.2 地图室（村庄功能）

地图室是玩家在世界中选择方向的战略界面：

```
┌──────────────────────────────────────────────────────┐
│                 世界地图（地图室）                      │
│                                                      │
│              [瘟疫地窖 Lv.3 ⚔️]                       │
│                    │                                 │
│  [圣堂废墟 Lv.1 ✓]─┼──[异端要塞 Lv.5 🔒]             │
│                    │                                 │
│              ★ 村庄 ★──[? 迷雾]                       │
│                                                      │
│  [出发补给] [选择塔] [派遣斥候] [返回村庄]            │
└──────────────────────────────────────────────────────┘
```

**地图室操作**：
- **查看塔**：点击已发现的塔，显示名称/等级/层级/特殊规则/推荐战力
- **派遣斥候**：消耗 12 信仰，斥候揭示一个方向的迷雾，可能发现新塔或套装情报
- **选择出发**：选塔后进入出发准备（分配骰子、装备、补给），消耗旅途补给进入

### 14.3 塔的身份

每座塔有**固定身份**，由塔 ID 决定，跨 Run 不变：

| 塔 ID 示例 | 主题 | 敌人族群 | 特殊规则 | BOSS 池 |
|------------|------|----------|----------|---------|
| 圣堂废墟 | 废弃教堂 | 堕落修士、圣像守卫 | 信仰获取翻倍，商店禁用 | 腐败审判官 |
| 瘟疫地窖 | 地下墓地 | 疫病鼠群、尸鬼 | 每 3 回合全体中毒 | 肿胀畸变体 |
| 异端要塞 | 军事堡垒 | 异端骑士、火焰法师 | 所有敌人护甲翻倍 | 异端大团长 |
| 沉默修道院 | 山中隐修 | 沉默僧侣、光之圣灵 | 无法使用信仰技能 | 圣母像·暗面 |
| 钟楼残骸 | 崩塌高塔 | 石像鬼、钟楼守卫 | 每层时间限制（20 回合） | 钟楼守护者 |

**塔身份定义**：
- 一座塔 = 一个主题 + 一个敌人池 + 1-2 条特殊规则 + 固定 BOSS
- 塔一旦发现就是**永久可进入的副本**
- 同一 Run 只进入一座塔

### 14.4 塔的等级与层级（正交维度）

塔的**等级**和**层级**是两个独立维度：

#### 塔等级（难度轴，固定）

| 塔等级 | 怪物属性倍率 | 掉落最低品质 | D100 出现率 | 推荐 |
|--------|-------------|-------------|------------|------|
| Lv.1 | ×0.8 | 白 | 0% | 新手 |
| Lv.2 | ×1.0 | 白 | 0% | 标准 |
| Lv.3 | ×1.3 | 蓝 | 0.1% | 困难 |
| Lv.4 | ×1.6 | 蓝+ | 0.5% | 噩梦 |
| Lv.5 | ×2.0 | 紫 | 2% | 地狱 |

- 塔等级是**固定的**（异端要塞永远是 Lv.5）
- 高等级塔需要先通关相邻低等级塔来**解锁进入权限**

#### 塔层级（深度，每次进入随机）

| 层级数 | 说明 | 出现概率 |
|--------|------|----------|
| 3 层 | 浅塔，速刷型 | 30% |
| 4-6 层 | 标准深度 | 50% |
| 7-9 层 | 深塔，远征型 | 15% |
| 10 层 | 极深塔，终局 | 5% |

- 层级数**每次进入塔时随机生成**，不受塔等级影响
- 层级多 ≠ 更难（低等级塔 10 层也是低级怪），但代表**探索时间长**
- 每层格子数参见 [7.1 棋盘结构](#71-棋盘结构)

### 14.5 塔内随机生成（每次进入）

| 要素 | 生成规则 |
|------|----------|
| 层级数 | 随机 3~10 层 |
| 每层格子数 | 参见 7.1 表格，随机微调 ±2 |
| 格子分布 | 按模板比例随机排列（战斗 40%/事件 20%/商店 10%/祈祷所 10%/宝藏 8%/陷阱 7%/BOSS 固定/上楼格固定） |
| BOSS 格 | 每层至少 1 个 BOSS 格 |
| 上楼格 | 每层至少 1 个上楼格（与 BOSS 格可能不同位置） |
| 稀有事件 | 概率 = 5% + 塔等级×3%（Lv.1=8%, Lv.5=20%） |

**生成锚点**：塔 ID（种子锚） + Run 种子 → 确定性随机（同种子同塔 = 相同布局）

### 14.6 塔的发现与解锁

| 方式 | 条件 | 说明 |
|------|------|------|
| 初始赠送 | 开局 | 村庄周围 2 座 Lv.1 塔 |
| 斥候探索 | 招募斥候 + 消耗 12 信仰 | 揭示一个方向，发现 1-2 座新塔 |
| 通关解锁 | 通关某座塔 | 该塔相邻的未发现塔自动揭示 |
| 特殊事件 | 村庄事件 / 传教士 | 低概率带回远方塔的情报 |

**塔等级递进解锁**：
- 通关任意 Lv.N 塔 → 解锁相邻 Lv.(N+1) 塔的进入权限
- 初始可进入 Lv.1 塔
- 未解锁的高等级塔在地图上显示为 🔒

### 14.7 旅途消耗

每次出发探索一座塔需要消耗**旅途补给**：

| 塔等级 | 旅途补给消耗 |
|--------|-------------|
| Lv.1-2 | 1 份补给 |
| Lv.3 | 2 份补给 |
| Lv.4 | 3 份补给 |
| Lv.5 | 5 份补给 |

- **补给来源**：村庄购买（1 金币 = 1 份）、Run 结算奖励
- 补给不足时无法进入高等级塔（避免一上来就送死）
- 轻装效果：补给消耗减 1（最低 1）

### 14.8 战斗奖励（三选一系统）

战斗胜利后，系统从奖励池中随机生成 **3 个选项**，玩家**必须选择 1 个**。奖励品质受**塔等级**影响：

#### 奖励池类型

| 类型 | 图标 | 说明 | 普通战概率 | 精英战概率 |
|------|------|------|-----------|-----------|
| **道具** | 🧪 | 消耗品（回复药、投掷物、增益卷轴等） | 30% | 15% |
| **装备** | 🗡️ | 随机品质的装备件 | 25% | 40% |
| **金币** | 💰 | 一笔金币奖励（10~50，随塔等级递增） | 25% | 15% |
| **骰子** | 🎲 | 随机品质骰子（新增奖励类型） | 10% | 15% |
| **状态恢复** | 💚 | HP 恢复 / 信仰恢复 / 诅咒清除等 | 10% | 5% |
| **遗物** | 📿 | 被动效果遗物（仅精英战/BOSS战） | 0% | 10% |

#### 品质权重（由塔等级决定）

| 塔等级 | 普通 | 精良 | 稀有 | 传说 |
|--------|------|------|------|------|
| Lv.1 | 70% | 25% | 5% | 0% |
| Lv.2 | 50% | 35% | 14% | 1% |
| Lv.3 | 30% | 35% | 30% | 5% |
| Lv.4 | 15% | 35% | 35% | 15% |
| Lv.5 | 5% | 25% | 40% | 30% |

#### 状态恢复细分

| 恢复类型 | 效果 | 出现条件 |
|----------|------|----------|
| 生命恢复 | 恢复 20%~40% 最大HP | 当前HP < 80% |
| 信仰恢复 | 恢复 3~5 点信仰值 | 当前信仰值 < 20 |
| 诅咒清除 | 降低 2~4 层诅咒 | 当前诅咒 ≥ 3 |
| 负重减轻 | 本层剩余移动中负重减半 | 当前负重 ≥ 80% |
| 全面休整 | 恢复 10%HP + 2信仰 + 1诅咒清除 | 随机低概率 |

### 14.9 层间结算

每层清完后进入「层间神坛」：

- 从 2 件装备/遗物/骰子中选 1 个
- 可选择「净化诅咒」（-5 金）或「强化一件装备」（-10 金）
- **村庄贡献选项**：将本层获得的部分资源捐给村庄（即时转化为村庄进度）
- **骰子磨损恢复**：所有骰子磨损 -2

### 14.10 通关与返回

- 击败塔顶 BOSS → 塔通关 → **获得塔通关奖励**（大量金币 + 信仰 + 稀有骰子）
- 可选择「返回村庄」或「在塔内继续绕圈刷资源」（每多一圈诅咒 +2）
- 同一 Run 可连续探索多座塔（需返回村庄 → 重新出发），但旅途补给有限

## 16. BOSS 设计

### 16.1 BOSS 机制通则

- 每座塔的**顶层固定一个 BOSS**，塔身份决定 BOSS 池（每座塔 1-2 个候选 BOSS，随机抽 1）
- BOSS 有专属像素立绘和战斗主题曲
- BOSS 有**阶段变化**：通常 2 个阶段
- BOSS 会根据**玩家装备套装**和**信仰值**调整策略：
  - 骑士套装满激活 → BOSS 强化对护甲的穿透
  - 高信仰 → BOSS 使用"亵渎攻击"克制圣洁
  - 低信仰 → BOSS 使用"秩序惩罚"攻击异端

### 16.2 BOSS 与塔对应

## 17. 肉鸽要素

### 17.1 每局随机内容

- **塔内层级数**：每次进入随机（3~10 层）
- **棋盘格子排布**：完全随机
- **怪物组合**：随机（含词缀系统），由塔身份限定族群范围
- **BOSS 二选一**：同一座塔的 BOSS 从候选池随机抽 1
- **道具/装备/骰子奖励**：随机（受塔等级影响品质）
- **遗物随机**：共约 60 个，每局约见到 15~20 个
- **事件随机**：共约 40 个，每局约触发 10~15 个
- **商店货架随机**

### 17.2 永久解锁（Meta 进度）

| 解锁条件 | 解锁内容 |
|----------|----------|
| 首次通关任意塔 | 地图室解锁；可派遣斥候探索 |
| 通关 Lv.3 塔 | 困难模式（高等级塔诅咒加速） |
| 村庄升至镇子 | 解锁稀有装备池 |
| 发现 10 座塔 | 特殊遗物「旅人之星」永久解锁（旅途补给消耗-1） |
| 获得全部信仰标签 | 真结局分支解锁 |
| 完成 10 次殉道者通关 | 特殊遗物「殉道者圣冠」永久解锁 |
| 村庄升至圣城 | 最终结局解锁（终极塔「神之眼」出现），特殊开场叙事 |

### 17.3 诅咒等级（Run 内难度调节）

| 诅咒层数 | 效果                         |
| -------- | ---------------------------- |
| 0-2      | 无附加效果                   |
| 3-5      | 怪物+10% HP                  |
| 6-8      | 商店价格 ×1.5                |
| 9-11     | 每回合棋盘随机移动 1 格      |
| 12+      | 每步移动额外触发随机负面事件 |

## 20. 死亡与复活机制

> v0.4 新增章节：明确死亡规则和复活流程。

### 20.1 死亡触发

- 玩家 HP 降为 0 时死亡
- 死亡**不会结束整个 Run**——玩家在村庄复活

### 20.2 死亡惩罚

| 惩罚项 | 说明 |
|--------|------|
| 金币掉落 | 失去**当前携带金币的 30%**，遗留在死亡格 |
| 信仰值掉落 | 失去**当前信仰值的 30%**，消散无法找回 |
| 装备掉落 | **小概率（15%）**掉落 1 件已装备的物品，遗留在死亡格 |
| 骰子碎裂 | 所有嵌入骰子的磨损值**直接拉满至碎裂**（词条失效，仍可移动，需铁匠铺或道具修复） |
| 进度保留 | 已贡献给村庄的资源**永久保留**；已通过的塔层**不需要重打** |

### 20.3 复活与拾取

```
死亡 → 自动传送回村庄
  ↓
村庄复活（HP 恢复至 50%，信仰恢复至基础值 5）
  ↓
重新出发 → 回到之前探索的塔
  ↓
到达死亡层 → 死亡格显示为「遗物标记」
  ↓
踩上死亡格 → 拾取掉落的金币和装备（信仰值无法找回）
```

**拾取规则**：
- 金币掉落物**永久存在**直到拾取
- 装备掉落物**只存在本次 Run**，如果再次死亡且未拾取则永久丢失
- 每次死亡会覆盖之前的掉落标记（只保留最近一次）

### 20.4 连续死亡保护

- 同一次 Run 内连续死亡 3 次，第 3 次死亡时**不掉落装备**（金币和信仰仍掉）
- 连续死亡会触发村庄 NPC 的对话安慰（叙事层面）
