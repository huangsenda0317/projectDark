# 04 · 垂直切片范围（可玩原型）

目标：在 **最小内容量** 下验证 **骰子棋盘 + 迷雾事件 + Boss 守门 + Roguelike 死亡** 的闭环，并能估算向 5–8 小时全量扩展的工程量。

---

## 1. 切片规模（锁定）

| 维度 | 垂直切片 | 全量目标（对照纲要） |
|------|----------|----------------------|
| 可玩层数 | **10 层**（层号 1–10，叙事上为第一章开头） | 99 层 / 三章 |
| 每层格子 | **30 格** 环形跑道 | 同左 |
| 上楼格 | 每层 **1** 个显性上楼 + **0–1** 个隐藏（切片内最多 1 个隐藏事件演示） | 1–2 |
| 迷雾 | 踩上揭晓；可选实现「邻格类型」一种难度 | 多种可选 |
| 骰子 | 1×d6；消耗品 **仅** 实装：`C13` 重掷、`C09` 定数 3 | 全表逐步 |
| 格子事件种类 | **15–20 种**（从 v0 表选取子集） | 28+ 扩展 |
| Boss 数量 | **3**（第 4、7、10 层守关） | 章内多节点 + 章末 |
| 消耗品实体 | **12** 种（表中前 12 或策划圈定） | 全文档 |
| 装备 | **武器 2、防具 2、饰品 2**（共 6 件可掉落） | 全套 |
| 魔法 | **3** 主动（`S01`、`S03`、再加 `S02` 或棋盘向 1 个） | 8+ |
| Meta | **1 条**解锁（如新事件进池）；**无**进阶 | 完整 meta |
| UI | 主菜单、棋盘、战斗 HUD、结算死亡屏、暂停 | 全章 |

## 3. 事件子集建议（从 01 文档选用）

优先实装：**EV01–EV07、EV08–EV12、EV16–EV19、EV21、EV24、EV27**（共 19 条），不足的用 `增益/害格/金币` 三条朴素占位事件补到 20。

---

## 4. 非目标（切片阶段刻意不做）

- 进阶（Ascension）与真结局分支。
- 第二章 / 第三章美术与事件池。
- 联网、成就、多语言（可仅占位）。
- 完整装备随机词条生成器（切片用手填 6 件）。

---

## 5. 验收标准（DoD）

1. **单 Run**：从第 1 层开始，可连续打完第 10 层 Boss 并显示胜利结算。
2. **死亡**：中途死亡返回主菜单，无存档或仅单槽 debug 存盘（按工程优先级）。
3. **棋盘**：平均每层踩 **≥8** 格仍可找到上楼格（可调随机种子 QA）。
4. **数据**：事件 / 道具为 **表驱动**（CSV/JSON），改表无需改码即可换参数。

---

## 6. 从切片到 EA 的扩展一步

- 将层数扩至 **33**（完整第一章），事件扩至 **≥25**，Boss 增至纲要中的 **章内 3 + 末 Boss** 配置，并接入 [05-进阶与存盘设计.md](05-进阶与存盘设计.md) 中的存盘节点。
