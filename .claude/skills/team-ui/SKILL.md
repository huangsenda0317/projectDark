# /team-ui

## Purpose

Orchestrate the UI team (ux-designer, ui-programmer, technical-artist, accessibility-specialist) to build UI features.

## Usage

```
/team-ui "inventory screen with drag-and-drop"
/team-ui "combat HUD with health, AP, and card hand"
```

## Workflow

### Phase 1 — UX Design (ux-designer)
- User flows and wireframes
- Interaction patterns
- Accessibility considerations

### Phase 2 — Visual Design (technical-artist)
- Layout and spacing
- Color and typography
- Animation and transitions

### Phase 3 — Architecture (ui-programmer)
- Scene structure
- Data binding approach
- Signal/event flow

### Phase 4 — Implementation (ui-programmer)
- Build UI scenes
- Wire up interactions
- Implement accessibility

### Phase 5 — Validation (accessibility-specialist + qa-tester)
- Keyboard/gamepad navigation
- Screen reader compatibility
- Visual regression

## Collaboration Protocol

- Wireframes approved before visual design
- Visual design approved before implementation
- Accessibility checked before sign-off
