# /brainstorm

## Purpose

Generate and evaluate game concepts using structured professional frameworks.

## When to Use

- Starting a new game from scratch
- Exploring alternative directions for an existing concept
- Validating whether a game idea has depth

## Workflow

### Phase 1: Discovery Questions

Ask the user 3-4 clarifying questions:

1. What genre or theme interests you? (Optional — leave blank for open exploration)
2. What excites you about games? (permadeath tension? build variety? skill mastery?)
3. Any reference games you love/hate and why?
4. Solo dev or team? (Affects scope recommendations)

### Phase 2: Generate Concepts

Generate 10 raw concept seeds across 3 categories:
- **Traditional** (combat-focused)
- **Hybrid** (combat + other verb)
- **Unconventional** (non-combat primary loop)

Each concept includes:
- One-sentence pitch
- Core verb (what the player DOES)
- MDA breakdown (Mechanics → Dynamics → Aesthetics)
- Reference games

### Phase 3: Deep Analysis

User selects 2-3 concepts. For each, perform:

1. **MDA Analysis**: Mechanics, Dynamics, Aesthetics
2. **SDT Mapping**: Autonomy, Competence, Relatedness
3. **Flow State Design**: Challenge-skill balance curve
4. **Bartle Targeting**: Which player types does this serve?
5. **Scope Assessment**: Realistic for solo/team?

### Phase 4: Recommendation

Present analysis and recommend one concept with reasoning. User makes the final decision.

### Phase 5: Formalize

Ask: "Should I create the full concept document now?"

If yes, delegate to `game-designer` to create `design/concept.md` using the template.

## Collaboration Protocol

- Never generate concepts without context
- Always present trade-offs, not just features
- Make recommendations but defer to user
- Get approval before writing files
