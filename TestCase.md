## 1. 主菜单与存档

### TC-MENU-01 新游戏启动（无种子）
- **优先级**：P0
- **前置**：无存档文件
- **步骤**：
  1. 删除 `user://mvp_save.json`（如存在）
  2. 启动游戏，进入主菜单
  3. SeedInput 为空，点击「新游戏」
- **预期**：
  - 跳转到村庄场景（`village.tscn`）
  - GameState 初始化为：HP=45, 金币=0, 信仰=10, 骰子=D4白+D6白, 骰子匣=2格

### TC-MENU-02 新游戏（输入数字种子）
- **优先级**：P1
- **前置**：无存档
- **步骤**：
  1. 在 SeedInput 输入 `12345`
  2. 点击「新游戏」
- **预期**：
  - `GameState.run_seed = 12345`
  - BoardHUD 显示 `[种子:12345]`

### TC-MENU-03 新游戏（输入文本种子）
- **优先级**：P1
- **前置**：无存档
- **步骤**：
  1. 在 SeedInput 输入 `hello`
  2. 点击「新游戏」
- **预期**：
  - `GameState.run_seed = hash("hello")`（固定值，非 -1）
  - 同文本种子每次棋盘布局一致

### TC-MENU-04 新游戏（种子为 0 或负数）
- **优先级**：P2
- **前置**：无存档
- **步骤**：
  1. 输入 `0`，点击「新游戏」→ 预期 seed=0（合法种子）
  2. 重开，输入 `-5`，点击「新游戏」→ 预期 seed=-5 或兜底随机

### TC-MENU-05 继续游戏按钮（无存档时）
- **优先级**：P1
- **前置**：删除 `user://mvp_save.json`
- **步骤**：进入主菜单
- **预期**：继续游戏按钮 **灰色不可点击**（`disabled = true`）

### TC-MENU-06 继续游戏按钮（有存档时）
- **优先级**：P0
- **前置**：已有存档文件（在棋盘上触发过 tile）
- **步骤**：重启游戏 → 进入主菜单 → 点击「继续游戏」
- **预期**：继续按钮可用，点击后恢复到存档所在的场景和状态

### TC-SAVE-01 自动存档触发
- **优先级**：P0
- **前置**：正常游戏，在棋盘上
- **步骤**：投掷骰子 → 移动 → 踩中任意格子触发事件
- **预期**：
  - `SaveManager.auto_save()` 被调用
  - `user://mvp_save.json` 文件被写入
  - JSON 包含 player/currencies/dice/equipment/inventory/run/death/village 全部 8 个 key

### TC-SAVE-02 装备正确序列化/反序列化
- **优先级**：P0
- **前置**：已将骑士重锤嵌入 D6，穿戴骑士胸甲
- **步骤**：
  1. 触发任意 tile 自动存档
  2. 退出游戏
  3. 重开 → 点击「继续游戏」
- **预期**：
  - 骑士重锤仍在武器槽，嵌入的 D6 仍存在
  - 骑士胸甲仍在胸甲槽
  - `apply_equipment_stats()` 和 `update_weight()` 正确执行

### TC-SAVE-03 背包内容正确序列化/反序列化
- **优先级**：P0
- **前置**：背包中有 healing_potion + reveal_crystal
- **步骤**：存档 → 退出 → 继续游戏
- **预期**：背包中 healing_potion 和 reveal_crystal 仍存在，数量和属性完整

### TC-SAVE-04 死亡掉落装备序列化
- **优先级**：P0
- **前置**：玩家死亡（death_equipment_dropped 非空）
- **步骤**：存档 → 退出 → 读档
- **预期**：
  - `GameState.death_equipment_dropped` 正确恢复
  - 装备的 embedded_dice 也正确恢复

### TC-SAVE-05 骰子匣容量序列化
- **优先级**：P1
- **前置**：打过 BOSS 且骰子匣扩容到 3 或 4
- **步骤**：存档 → 退出 → 读档
- **预期**：`GameState.dice_box_capacity` 恢复为扩容后的值

### TC-SAVE-06 命运值序列化
- **优先级**：P1
- **前置**：使用过命运骰（destiny_value != -1）
- **步骤**：存档 → 退出 → 读档
- **预期**：`GameState.destiny_value` 恢复正确

### TC-SAVE-07 存档文件不存在时 load_game 安全返回
- **优先级**：P2
- **前置**：删除存档文件
- **步骤**：调用 `SaveManager.load_game()`
- **预期**：返回 `false`，不崩溃，不修改 GameState

## 2. 环形棋盘

### TC-BOARD-01 初始棋盘生成
- **优先级**：P0
- **步骤**：从村庄出发 → 进入圣堂废墟第 1 层
- **预期**：
  - 生成 18-22 格环形棋盘
  - 玩家 token 在第 0 格
  - 第 0 格已揭示，其余灰色/不可见

### TC-BOARD-02 顺时针移动
- **优先级**：P0
- **步骤**：投掷骰子 → 选择顺时针方向
- **预期**：
  - 棋子沿环顺时针滑动到目标格
  - 目标格自动揭示
  - 经过的中间格子不自动触发

### TC-BOARD-03 逆时针移动
- **优先级**：P0
- **步骤**：投掷骰子 → 选择逆时针方向
- **预期**：棋子沿环逆时针滑动到目标格，目标格揭示

### TC-BOARD-04 格子点击（相邻格）
- **优先级**：P1
- **步骤**：点击当前所在格相邻（顺时针或逆时针方向 1 步内）的格子
- **预期**：提示距离，不触发移动

### TC-BOARD-05 格子点击（远距离）
- **优先级**：P1
- **步骤**：点击距离 > 1 的格子
- **预期**：提示距离（不移动，需要投掷骰子）

### TC-BOARD-06 套圈检测
- **优先级**：P1
- **前置**：位于第 5 格，棋盘共 20 格，lap_start_index=0
- **步骤**：投掷骰子走 16 步以上越过 index=0
- **预期**：
  - 触发射 `lap_completed` 信号
  - 诅咒层数 +1
  - 已触发格变为「虚空格」

### TC-BOARD-07 套圈后虚空格转换
- **优先级**：P1
- **前置**：完成至少一圈，部分格子已触发
- **步骤**：观察已触发格
- **预期**：已触发格变为「虚空格」（tile_type="void"），外观变化

### TC-BOARD-08 上楼格触发
- **优先级**：P0
- **步骤**：踩中上楼格（tile_type="stair"）
- **预期**：
  - 层数 +1（`GameState.current_floor += 1`）
  - 所有骰子磨损 -2
  - 新棋盘生成

### TC-BOARD-09 最后一层上楼
- **优先级**：P1
- **前置**：`current_floor >= total_floors_in_tower`
- **步骤**：踩中上楼格
- **预期**：触发 `run_ended` → 跳转村庄场景

### TC-BOARD-10 方向指示器
- **优先级**：P2
- **步骤**：投掷骰子后，观察目标格
- **预期**：目标格显示方向指示标记

### TC-BOARD-11 地板层治愈之光
- **优先级**：P1
- **前置**：拥有一颗带「治愈之光」词条的骰子
- **步骤**：进入新层（`_generate_floor()`）
- **预期**：HP 恢复 3 点

## 3. 骰子系统

### TC-DICE-01 初始骰子
- **优先级**：P0
- **步骤**：开始新游戏 → 查看骰子匣
- **预期**：
  - 骰子匣中有 1×D4（白色）+ 1×D6（白色）
  - 骰子匣容量显示 2 格

### TC-DICE-02 骰子面数范围
- **优先级**：P1
- **步骤**：在商店/战斗掉落中获取 D4/D6/D8/D10/D12 骰子
- **预期**：
  - D4：投掷 1-4
  - D6：投掷 1-6
  - D8：投掷 1-8
  - D10：投掷 1-10
  - D12：投掷 1-12

### TC-DICE-03 骰子品质词条数
- **优先级**：P1
- **步骤**：获取白/蓝/紫品质骰子
- **预期**：白色=1 词条，蓝色=2 词条，紫色=3 词条

### TC-DICE-04 骰子嵌入（正常流程）
- **优先级**：P0
- **步骤**：
  1. 非战斗状态打开骰子分配面板
  2. 将 D6 嵌入武器槽
- **预期**：
  - 消耗信仰 = `6 ÷ 3 = 2` 点
  - 骰子状态变为「嵌入」
  - 武器伤害公式激活（`1d6+...`）

### TC-DICE-05 骰子嵌入（战斗状态锁定）
- **优先级**：P1
- **前置**：踩到战斗格，战斗开始
- **步骤**：战斗期间尝试打开分配面板或重分配骰子
- **预期**：分配操作被拒绝或界面不可用

### TC-DICE-06 嵌入骰子参与移动投掷
- **优先级**：P0
- **前置**：D6 已嵌入武器
- **步骤**：在移动投掷时选择嵌入中的 D6 投掷
- **预期**：
  - D6 正常投掷，点数=步数
  - D6 **不脱离武器**，仍保持嵌入状态

### TC-DICE-07 取回嵌入骰子
- **优先级**：P1
- **前置**：D6 嵌入武器
- **步骤**：非战斗状态，将 D6 从武器取回
- **预期**：
  - 骰子回到自由状态
  - **不退还**已消耗的信仰值
  - 武器伤害公式失效（基础伤害降为 1 或武器裸值）

### TC-DICE-08 骰子匣满时获取骰子
- **优先级**：P1
- **前置**：骰子匣已满（容量 2，已持有 2 枚）
- **步骤**：战斗掉落/商店获取新骰子
- **预期**：提示「骰子匣已满，无法拾取」，骰子不加入

### TC-DICE-09 骰子匣扩容
- **优先级**：P1
- **步骤**：击败 BOSS → 低概率获得扩容奖励
- **预期**：`dice_box_capacity` 增加 1（MVP 上限 4），提示扩容成功

### TC-DICE-10 兜底 D4 领取
- **优先级**：P2
- **前置**：所有骰子碎裂，骰子匣为空
- **步骤**：返回村庄 → 向 NPC 领取兜底骰子
- **预期**：获得 1×D4 白色骰子（免费）

### TC-DICE-11 投掷词条——步数修正
- **优先级**：P1
- **前置**：持有带「步数修正」词条的骰子
- **步骤**：投掷该骰子，选择 ±1 修正
- **预期**：最终移动步数 = 原点数 ± 1

## 4. 格子类型

### TC-TILE-01 战斗格
- **优先级**：P0
- **步骤**：踩中战斗格
- **预期**：
  - 进入战斗状态（`State.COMBAT`）
  - 骰子分配锁定
  - 战斗场景显示

### TC-TILE-02 商店格
- **优先级**：P0
- **步骤**：踩中商店格
- **预期**：
  - 进入购物状态（`State.SHOPPING`）
  - 商店面板显示货架（骰子/道具/装备）
  - 可用金币购买，关闭后返回 IDLE

### TC-TILE-03 祈祷所
- **优先级**：P0
- **步骤**：踩中祈祷所（prayer）
- **预期**：
  - 获得 2-4 点信仰值（`RNG.randi_range(2, 4)`）
  - Toast 提示「祈祷所 +N 信仰」

### TC-TILE-04 宝藏格
- **优先级**：P0
- **步骤**：踩中宝藏格
- **预期**：
  - 进入宝藏状态（`State.TREASURE`）
  - 宝藏面板显示随机奖励（装备/道具/金币）
  - 选择或确认后关闭

### TC-TILE-05 事件格
- **优先级**：P0
- **步骤**：踩中事件格
- **预期**：
  - 进入事件状态（`State.EVENT`）
  - 事件面板显示文本 + 2 个选项
  - 选择后结算效果（金币/信仰/HP/装备变化）

### TC-TILE-06 陷阱格
- **优先级**：P0
- **步骤**：踩中陷阱格
- **预期**：
  - 受到 3-8 点伤害（`RNG.randi_range(3, 8)`）
  - Toast 提示「陷阱! -N HP」
  - 不打开面板，直接结算

### TC-TILE-07 BOSS 格
- **优先级**：P0
- **步骤**：踩中 BOSS 格
- **预期**：
  - 进入战斗状态
  - 敌人为 BOSS（`is_boss=true`）
  - 击败后触发 `boss_defeated` 奖励

### TC-TILE-08 虚空格（第二圈）
- **优先级**：P1
- **前置**：完成一圈，该格已被触发
- **步骤**：踩中已变为虚空格的格子
- **预期**：跳过，无事件触发，诅咒 +1

## 5. 战斗系统

### TC-CBT-01 战斗初始化
- **优先级**：P0
- **步骤**：踩战斗格 → 战斗开始
- **预期**：
  - 创建 PlayerEntity 和 EnemyEntity
  - AP = `GameState.get_ap_per_turn()`（基础 3）
  - 状态为 `PLAYER_TURN`

### TC-CBT-02 普攻（消耗 1 AP）
- **优先级**：P0
- **前置**：武器嵌入骰子（如 D6 嵌入长剑）
- **步骤**：执行 `player_attack()`
- **预期**：
  - AP -1
  - 伤害值 = 武器骰子公式计算结果（`1d6+Z`）
  - 敌人 HP 减少
  - 磨损累积（嵌入骰子 wear+1，每场战斗仅 +1）

### TC-CBT-03 普攻（无武器/无骰子）
- **优先级**：P1
- **前置**：武器槽为空或无嵌入骰子
- **步骤**：执行 `player_attack()`
- **预期**：伤害=1（赤手空拳），AP-1

### TC-CBT-04 防御（消耗 1 AP）
- **优先级**：P0
- **前置**：胸甲嵌入骰子
- **步骤**：执行 `player_defend()`
- **预期**：
  - AP -1
  - 获得护盾值 = `骰子面数`（胸甲）+ `骰子面数 × 格挡系数`（副手，如有）
  - 磨损累积

### TC-CBT-05 技能（消耗 2 AP）
- **优先级**：P0
- **前置**：穿着 4 件骑士套装（解锁神圣冲锋）
- **步骤**：执行 `player_use_skill("holy_charge")`
- **预期**：
  - AP -2
  - 伤害 = `STR×3 + 武器嵌入骰子面数`
  - 套装件数不足时 blocked=true

### TC-CBT-06 道具使用（消耗 1 AP）
- **优先级**：P1
- **前置**：背包中有消耗品
- **步骤**：执行 `player_use_item(item)`
- **预期**：AP-1，道具效果生效

### TC-CBT-07 AP 耗尽自动回合结束
- **优先级**：P0
- **步骤**：连续攻击直至 AP=0（或无法支付行动）
- **预期**：`_check_turn_end()` 触发 → 状态变为 `ENEMY_TURN`

### TC-CBT-08 敌意阶段——攻击意图
- **优先级**：P1
- **前置**：敌人回合，意图为 attack
- **步骤**：观察敌人意图显示
- **预期**：显示 ⚔️ 攻击图标，execute 后玩家受对应伤害

### TC-CBT-09 敌意阶段——防御意图
- **优先级**：P1
- **前置**：敌人回合，意图为 defend
- **步骤**：观察敌人意图显示
- **预期**：显示 🛡️ 图标，execute 后敌人获得护盾

### TC-CBT-10 敌意阶段——强力攻击意图
- **优先级**：P1
- **前置**：敌人回合，意图为 heavy_attack
- **步骤**：观察敌人意图显示
- **预期**：显示 🔥 图标，execute 后玩家受高额伤害

### TC-CBT-11 战斗胜利
- **优先级**：P0
- **步骤**：将敌人 HP 降至 0
- **预期**：
  - 触发 `combat_victory` 信号
  - 获得经验/金币/骰子掉落
  - 战斗面板关闭，返回棋盘 IDLE

### TC-CBT-12 战斗失败
- **优先级**：P0
- **步骤**：玩家 HP 降至 0
- **预期**：
  - 触发 `combat_defeat` 信号
  - 显示「战败...返回村庄」
  - 2 秒后跳转村庄场景
  - 触发死亡惩罚

### TC-CBT-13 磨损——每场战斗每颗骰子仅 +1
- **优先级**：P0
- **步骤**：
  1. 用嵌入 D6 的武器攻击 3 次（同一场战斗）
  2. 检查 D6 的磨损值
- **预期**：磨损值 = 1（不是 3）

### TC-CBT-14 磨损——多装备分别累积
- **优先级**：P1
- **前置**：武器嵌入 D6，胸甲嵌入 D8
- **步骤**：攻击 1 次 + 防御 1 次
- **预期**：D6 磨损+1（武器攻击），D8 磨损+1（胸甲防御）

### TC-CBT-15 磨损——碎裂
- **优先级**：P0
- **前置**：D6 磨损 = 5（max_wear=6）
- **步骤**：再次攻击 1 次（磨损→6）
- **预期**：
  - D6 `is_shattered = true`
  - 词条失效、嵌入效果无效
  - D6 仍可投掷移动

### TC-CBT-16 磨损——楼层恢复
- **优先级**：P0
- **前置**：D6 磨损 = 3
- **步骤**：踩上楼格进入下一层
- **预期**：D6 磨损 = 1（wear_recovery_per_floor = 2）
